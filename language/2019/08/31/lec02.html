C++ 01 - 변수 (2)

by Gnues

2019. 8. 31. 20:57
in Language

그간 사용해오던 언어들을 다시 정리하고 애매모호한 내용들을 바로잡기 위한 글입니다.

축소하지 않는 초기화

long l2 = 1234567890123;

long 타입이 64비트인 아키텍쳐의 경우 위 코드는 문제없이 잘 컴파일 될 것이다.

하지만 long의 크기가 32비트인 경우에는 위 리터럴 값은 할당하기에 너무 크다.

컴파일은 가능하지만 경고 메시지와 함께 앞쪽 비트가 버려지게 될 것이다.

C++11

C++11은 데이터가 손실되지 않음(값들이 축소되지않음, Not Narrowed)을 확인하는 초기화를 도입했다.

유니폼 초기화, 중괄호 초기화가 존재한다.

long l = { 1234567890123 };

중괄호 안에 있는 값들은 축소할 수 없으며 컴파일러는 변수 l이 타겟 아키텍쳐에서 갑을 저장할 수 있는지 확인할 것이다.

컴파일러의 축소 보호는 값을 초기화할 때 정밀도를 잃지 않는다는 것을 확인시켜준다.

즉, 묵시적 변환을 사용하지 않은 채 값을 그대로 저장할 수 없다면 에러를 유발시킨다.

int i1 = 3.14     // warning C4244:  '초기화 중': 'double'에서 'int'(으)로 변환하면서 데이터가 손실될 수 있습니다.
int i2 = { 3.14 } // error C2397:  'double'에서 'int'(으)로의 변환에는 축소 변환이 필요합니다.


unsigned u2 = -3;     // 컴파일러 가능 (위험)
unsigned u3 = { -3 }; // error C2397:  'int'에서 'unsigned int'(으)로의 변환에는 축소 변환이 필요합니다.

리터럴을 사용해서 초기화 할 때는 해당 리터럴 값이 해당 타입에서 표현할 수 있는지를 검사한다.

float f1 = { 3.14 }; // warning C4305:  '초기화 중': 'double'에서 'float'(으)로 잘립니다.

double 타입의 3.14 리터럴 값은 모든 이진 부동 소수점 타입으로 나타낼 수는 없지만, 컴파일러는 3.14에 가장 가까운 값으로 f1을 설정한다.

특정 타입 변수나 (리터럴이 아닌) 상수에서 축소 보호 초기화를 할 때 모든 값들이 손실 없는 방식으로 변환가능한지 고려해야 한다.

double d1 = 3.14;
float f2 = { d1 };      // error C2397:  'double'에서 'float'(으)로의 변환에는 축소 변환이 필요합니다.

unsigned u2 = { 3 };
int i3 = { 2 };
unsigned u3 = { i3 };   // error C2397:  'int'에서 'unsigned int'(으)로의 변환에는 축소 변환이 필요합니다.
int i4 = { u2 };        // error C2397:  'unsigned int'에서 'int'(으)로의 변환에는 축소 변환이 필요합니다.

범위

범위(Scope)는 변수, 상수의 수명과 가시성을 결정한다.

모든 변수를 사용 하려면 사용 전에 타입 지정자와 함께 선언해야 한다.

하나의 변수는 전역 범위(global scope)나 지역 범위(local scope)에 위치할 수 있다.

전역 정의

전역 변수는 모든 함수의 바깥에 선언한다.

전역 변수를 선언하고 나면 코드 어디에서나(함수 안에서도) 사용가능하다.

변수를 쉽게 사용할 수 있지만, 프로그램의 규모가 커짐에 따라 전역 변수들을 추적하기가 어려워진다.

때문에 가능하면 전역 변수 사용을 지양하는 것이 좋다.

int global_var = 10;
int main()
{
  std::cout << global_var << std::endl;
  ...
}

지역 정의

지역 변수는 함수 안에서 선언한다.

해당 지역 변수의 수명은 {...}로 둘러싸인 블록으로 제한된다.

블록 외부(Out of scope)에서 지역 변수를 접근할 수 없다.

void Scope()
{
  const double pi = 3.14159265358979323846264338327950288419716939;
  std::cout << pi << std::endl;

  {
    const double pi2 = 3.14159265358979323846264338327950288419716939;
  }
  std::cout << pi2 << std::endl;  // error C2065:  'pi2': 선언되지 않은 식별자입니다.
}
std::cout << pi << std::endl;   // error C2065:  'pi': 선언되지 않은 식별자입니다.

숨기기

같은 이름의 변수가 중첩된 범위에 있으면 오직 하나의 변수만 접근할 수 있다.

안쪽 범위(가장 내부에 선언된 블록)에 있는 변수는 바깥 범위에 선언된 동일한 이름의 변수를 숨긴다.

void Hide()
{
  int a = 1;  // a#1
  {
    std::cout << a << std::endl;  // a#1: 1
    int a = 2;                    // a#2 (a#1를 숨김)
    {
      std::cout << a << std::endl;  // a#2: 2
      int a = 3;                    // a#3 (a#2를 숨김)
      std::cout << a << std::endl;  // a#3: 3
      {
        a = 11;                     // a#3: 11
      }
      std::cout << a << std::endl;  // a#3: 11
    } // a#3의 수명은 여기까지
    std::cout << a << std::endl;    // a#2: 2 (a#3의 수명이 끝나고 다시 a#2를 바라봄)
  } // a#2의 수명은 여기까지
  std::cout << a << std::endl;  // a#1: 1
}

각 변수의 가시성은 수명에서 숨겨질 때의 시간을 뺀 값이다.

동일한 범위에 동일한 변수 이름을 두 번 이상 정의하면 오류가 발생한다.

int a = 1;
int a = 2; // error C2374:  'a': 재정의. 여러 번 초기화했습니다.

범위의 장점은 변수가 범위 바깥 어딘가에 이미 정의되었는지 걱정할 필요가 없다는 것이다.

앞서 정의된 변수는 숨겨지고 이로 인해 충돌을 피할 수 있지만 숨겨진 변수는 접근할 수가 없다.

이름을 바꿈으로써 어느정도 해결할 수 있지만 네임 스페이스(Namespace)를 사용하는 것이 더 현명한 방법이다.

static 변수는 이 규칙을 따르지 않는다.
