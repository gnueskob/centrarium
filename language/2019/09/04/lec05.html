C++ 01 - 연산자 (2)

by Gnues

2019. 9. 4. 22:42
in Language

그간 사용해오던 언어들을 다시 정리하고 애매모호한 내용들을 바로잡기 위한 글입니다.

불 연산자

논리 연산자와 관계 연산자가 존재하며 전부 bool 타입을 반환한다.

연산 표현식
Not !x
   
크다 x > y
크거나 같다 x >= y
   
작다 x < y
작거나 같다 x <= y
   
같다 x == y
같지 않다 x != y
   
논리 AND x && y
논리 OR x || y

위 표는 우선 순위별로 정렬되어있다.

이진 관계, 논리 연산자는 산술 연산자보다 우선순위가 낮다.

4 >= 1 + 7의 표현식은 4 >= (1 + 7)로 처리된다.

반대로, 논리 부정 단항 연산자 !는 모든 이항 연산자보다 먼저 처리된다.

비트 연산자

비트 연산자로 정수 탕비의 단일 비트를 검사하거나 조작할 수 있다.

연산 표현식
NOT(1의 보수) ~x
왼쪽 시프트 x « y
오른쪽 시프트 x » y
비트 AND x & y
비트 OR x | y
비트 XOR x ^ y

위 표는 우선 순위별로 정렬되어있다.

비트 연산자는 보통 시스템 프로그래밍이나 효율적인 알고리즘적인 요소에 주로 사용된다.

할당 연산자

Lvalue의 값은 할당 연산자를 통해 설정할 수 있다.

object = expr;

object의 타입과 expr의 타입이 서로 일치하지 않을 때, 가능하면 expr의 타입을 강제 변환해서 할당하게 된다.

할당 연산자는 오른쪽에서 왼쪽 순서(right-associative)로 결합한다.

obj3 = obj2 = obj1 = expr;  // obj3 = (obj2 = (obj1 = expr)));

또한 할당 연산자는 왼쪽 Lvalue를 반환한다.

(obj = 3) = 2;  // OK, obj: 2

합성 할당 연산자

합성 할당 연산자들은 오른쪽에 있는 인수로 왼쪽의 Lvalue에게 산술 연산이나 비트 연산을 적용한다.

a += b; // a = a + b;

모든 할당 연산자는 모든 산술, 비트 연산자보다 더 낮은 우선순위를 가지므로 오른쪽에 있는 표현식은 항상 합성 할당 연산자를 수행하기 전에 먼저 계산된다.

a *= b + c - d; // a = a * (b + c - d);
연산 표현식
단순 할당 x = y
곱셈 후 할당 x *= y
나눗셈 후 할당 x /= y
나머지 후 할당 x %= y
덧셈 후 할당 x += y
뺄셈 후 할당 x -= y
왼쪽 시프트 후 할당 x «= y
오른쪽 시프트 후 할당 x »= y
AND 후 할당 x &= y
OR 후 할당 x |= y
XOR 후 할당 x ^= y

위 표는 우선 순위별로 정렬되어있다.
