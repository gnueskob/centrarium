C++ 01 - 연산자 (1)

by Gnues

2019. 9. 2. 23:57
in Language

그간 사용해오던 언어들을 다시 정리하고 애매모호한 내용들을 바로잡기 위한 글입니다.

C++에 내장된 연산자들은 다음과 같다.

  • 계산연산자
    • 산술 연산자: ++, --, *, % …
    • 불 연산자
      • 비교 연산자: <=, != …
      • 논리 연산자: &&, ||
    • 비트 연산자: ~, <<, >>, & …
    • 할당 연산자: =, += …
    • 프로그램 흐름 연산자: 함수호출, ?:, |
    • 메모리 처리 연산자: new, delete
    • 접근 연산자: ., ->, [ ], * …
    • 타입 처리 연산자: dynamic_cast, typeid, sizeof, alignof …
    • 오류 처리 연산자: throw

일부 연산자는 적절한 언어 기능과 함께 설명하는 것이 더 이해하기 쉬우므로 넘어간다.

대부분의 연산자는 사용자 정의 타입에서 오버로드할 수 있다.

산술 연산자

연산 표현식
후위 증가 x++
후위 감소 x–
   
전위 증가 ++x
전위 감소 –x
단항 덧셈 +x
단항 뺄셈 -x
   
곱셈 x * y
나눗셈 x / y
나머지 x % y
   
이항 덧셈 x + y
이항 뺄셈 x - y

위 표는 우선 순위별로 정렬되어있다.

증감 연산자

증감 연산은 수정 가능하고 주소를 지정할 수 있는 Lvalue에만 적용가능하다.

int i = 3;  // i: Lvalue, 3: Rvalue
i++;        // OK
(3+6)++;    // Error

전위 증감 연산자는 수정된 값을 반환하고, 후위 증감 연산자는 기존 값을 반환한다.

int i = 3, j = 3;
int k = i++ + 4;  // i: 4, k: 7
int l = ++j + 4;  // j: 4, l: 8

더 정확하게 말하자면, 전위 증감 연산자는 수정된 Lvalue를 반환하고 후위 증감 연산자는 기존 값을 갖는 Rvalue를 반환한다.

++i = 1;  // OK
i++ = 2;  // error C2106:  '=': 왼쪽 피연산자는 l-value이어야 합니다.

// 후의 증감 연산자가 더 우선순위가 높음
// 때문에 i-- 연산으로 먼저 Rvalue를 반환
// ++(i--) 연산은 ++(Rvalue)와 같으므로 컴파일 에러
++i--;    // error C2105:  '++'에 l-value가 필요합니다.

때문에 증감 연산자는 다른 연산 수식에 함께 사용하는 것을 지양해야 한다.

수식 자체가 사이드 이펙트를 갖지 않아야 이해하기 쉽고 컴파일러가 최적화하기도 쉽다.

단항 연산자

단항 연산자는 일반 숫자에 적용시켜 부호를 바꿀수 있는 수준으로 표준 타입에는 아무런 영향을 주지 않는다.

후위 증감 연산자와 마찬가지로 Rvalue를 반환한다.

int i = 3;
int j = -i; // j: -3
+i = 1;     // error C2106:  '=': 왼쪽 피연산자는 l-value이어야 합니다.

다만 사용자 지정 타입에서 단항 연산자를 따로 정의하여 해당 타입에 대한 덧셈, 뺄셈 동작을 정의할 수 있다.

일반적으로 단항 연산자가 이항 연산자보다 우선 순위가 높다.

[Modern C++ 입문, 피터고츠슐링]

접두사, 접미사에 단항 표기법을 모두 적용한 경우 접두사의 단항 연산자가 더 우선순위가 높다

전위, 후위 증감 연산자도 단항 연산자일텐데 후위가 먼저 적용됨(?)

곱셈, 나눗셈, 나머지

*, / 연산자는 모든 숫자 타입에 정의되어 있다.

나눗셈은 정수끼리 수행될 때 소수부분을 버린다는 것을 숙지하자.

% 연산자는 정수 나눗셈의 나머지를 산출하므로 두 인수 모두 정수형이여야 한다.

일반 수학에서 처럼 사칙연산 순서가 동일하게 적용된다.

수식에 곱셈, 나눗셈을 먼저 처리하고 덧셈, 뺄셈을 처리한다.

연산은 왼쪽에서 오른쪽 순서로 결합(left-associative)한다.

중요한 것은 인수들의 계산 순서는 정의되지 않았다는 것이다.

int i = 3, j = 4;
int k = f(++i) + g(++i) + j;

위 수식에서 f(++i) + g(++i) 덧셈이 g(++i) + j 덧셈보다 먼저 계산될 것은 분명하다.

하지만 f(++i)와 g(++i) 중 어느 표현식이 먼저 계산될 것인지는 컴파일러 구현에 따라 다르다.

이항 연산의 두 인수가 서로 다른 타입을 가질 때에는 하나 혹은 두 인수가 규칙에 따라 타입이 자동 변환된다.

이러한 변환은 정밀도 손실을 초래할 수 있다.

long long l = 123456789012;       // 123456789012
long long l2 = l + 1.0f - 1.0;    // 123456790527
long long l3 = l + (1.0f - 1.0);  // 123456789012

l2의 경우 l + 1.0f에 의해서 중간 변환으로 float 타입으로 변환되고 1.23456791e+11값을 갖게된다.

이 단계에서 정밀도 손실이 발생되기 때문에 (double) 1.0값을 빼도 원래 값으로 복원되지 않는다.
